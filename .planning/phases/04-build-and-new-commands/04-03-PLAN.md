**configure():**

**validate(definition):**
This is the non-interactive path (used by tests).

1. Call `inferTypeAndName(process.argv)` to get `typeArg` and `nameArg`
2. If typeArg is undefined or nameArg is undefined: throw/panic because non-interactive mode requires positional args
3. Call `inferTemplate(process.argv, !!nameArg)` to get `templateId`. Default to 'qwik' if undefined.
4. rootDir = `process.cwd()`
5. Return `{ typeArg, nameArg, templateId, rootDir }` (cast typeArg to TemplateType)

**interact(definition):**
This is the interactive path (when TTY is available).

1. Call `inferTypeAndName(process.argv)` to get initial typeArg and nameArg
2. If typeArg is undefined: prompt with `this.scanChoice("What type?", [...])` for route/component/markdown/mdx
3. If nameArg is undefined: prompt with `this.scanString("Name:")` -- then re-infer type from the name if it starts with `/`
4. Call `inferTemplate(process.argv, !!nameArg)` for templateId
5. If templateId is undefined: load templates, filter by typeArg, if exactly 1 -> auto-select (NEW-06), else prompt
6. rootDir = `process.cwd()`
7. Return `{ typeArg, nameArg, templateId, rootDir }`

**execute(input):**
1. Load templates via `loadTemplates()`
2. Find the template matching `input.templateId`
3. Get template files for `input.typeArg` from the template
4. If no template files found: panic with error message
5. Determine the name segment for parseInputName:
   - For routes/markdown/mdx: use the LAST path segment of nameArg (e.g., `/dashboard` -> `dashboard`, `/blog/post` -> `post`)
   - For components: use the full nameArg (e.g., `counter`)
6. Call `parseInputName(nameSegment)` to get `{ slug, name }`
7. Call `getOutDir(input.rootDir, input.typeArg, input.nameArg)` to get outDir
8. For each template file: call `writeTemplateFile(outDir, templateFile, slug, name)`
9. Log success message (e.g., using console.log or clack note)
10. Return 0

**Error handling:**
- Wrap execute() body in try/catch
- If the error message contains "already exists": log the error message to stderr/stdout and return 1 (NEW-04)
- Other errors: panic or return 1
