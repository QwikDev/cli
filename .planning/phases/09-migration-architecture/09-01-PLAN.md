    Implement `detectInstalledVersion(rootDir: string): string | null`:
    - Read `package.json` from `rootDir` using `readFileSync` + `JSON.parse`

    Export `MigrationStep` interface:
    ```typescript
    export interface MigrationStep {
      version: string;   // semver version this migration brings the project to (e.g. "2.0.0")
      label: string;     // human-readable label (e.g. "v2")
      run: (rootDir: string) => Promise<void>;
    }
    ```

    Implement `buildMigrationChain(fromVersion: string | null, toVersion: string, registry: MigrationStep[]): MigrationStep[]`:
    - If `fromVersion` is `null` (no Qwik detected), return ALL registry steps (run everything)
    - Use `semver.coerce(fromVersion)` to normalize range strings like `^2.1.0` to `2.1.0`
    - If `semver.coerce()` returns `null` (malformed version like `*` or `workspace:*`), return ALL steps
    - Filter registry: return only steps where `semver.gt(step.version, coerced.version)` is `true`
    - The registry is already ordered — no sorting needed

    Import `semver` (already installed at `^7.7.4`).

    **Step 4 — Create src/upgrade/orchestrator.ts:**

    Define `MIGRATION_REGISTRY` array with the v2 migration step:
    ```typescript
    import { runV2Migration } from "../../migrations/v2/index.js";
    // ... other imports
    export const MIGRATION_REGISTRY: MigrationStep[] = [
      { version: "2.0.0", label: "v2", run: runV2Migration },
    ];
    ```

    Implement `getLatestQwikVersion(): Promise<string | null>`:
    - Wraps `getLatestV2Version("@qwik.dev/core")` from `../../migrations/v2/versions.js`
    - Returns the result directly

    Implement `runUpgrade(rootDir: string): Promise<void>`:
    1. Call `detectInstalledVersion(rootDir)` to get installed version
    2. Call `getLatestQwikVersion()` to get latest version from npm
    3. If latest is null, log warning "could not resolve latest version from registry" and use `"2.0.0"` as fallback target
    4. Call `buildMigrationChain(installedRaw, latestVersion ?? "2.0.0", MIGRATION_REGISTRY)`
    5. If chain is empty: `console.log("Already up to date.")` and return (the no-op path — MIGR-05)
    6. Otherwise: iterate chain sequentially, calling `await step.run(rootDir)` for each step, logging `Running migration: ${step.label}` before each
    7. After all steps: check latest version again and if installed is still behind, call `updateDependencies()` to install latest Qwik packages (MIGR-02 — dependency update is separate from migration)

    **IMPORTANT:** The relative import path from `src/upgrade/orchestrator.ts` to `migrations/v2/index.ts` is `../../migrations/v2/index.js`. This works because tsdown bundles from `src/` entry points and follows relative paths outside `src/`. No new tsdown entry point is needed — the orchestrator is reachable via `src/router.ts` -> `src/commands/migrate/index.ts` -> `src/upgrade/orchestrator.ts`.
