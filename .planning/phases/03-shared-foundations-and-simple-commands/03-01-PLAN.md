Replace the stub `execute()` in `src/commands/version/index.ts` with the dual-path version resolution pattern from research.

Add these imports at top:
```typescript
import { readFileSync } from "node:fs";
import { dirname, join } from "node:path";
import { fileURLToPath } from "node:url";
```

Add a module-level helper function `readPackageVersion()`:
- Uses `fileURLToPath(import.meta.url)` to get the current file path
- Resolves `join(dirname(__filename), "..", "..", "..", "package.json")` — 3 levels up from `src/commands/version/index.ts` to project root
- Reads with `readFileSync(pkgPath, "utf-8")`, JSON.parse, returns `.version`

Update `execute()`:
```typescript
protected async execute(_input: VersionInput): Promise<number> {
  let version: string;
  try {
    version = QWIK_VERSION;
  } catch {
    version = readPackageVersion();
  }
  console.log(version);
  return 0;
}
```

The `QWIK_VERSION` global is already declared in `src/types.ts`. At build time tsdown replaces it with the literal string. In tsx test mode it throws ReferenceError, caught by the try/catch, falling back to package.json read.

Do NOT add `declare const QWIK_VERSION` in this file — it is already in `src/types.ts`.
Do NOT use `process.env.QWIK_VERSION` — the injection is via `define` (compile-time replacement), not env var.
Do NOT call `process.exit()` — the router handles exit codes from the return value.
