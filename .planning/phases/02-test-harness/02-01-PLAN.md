Create `tests/integration/helpers/cli.ts` with:
- `CliResult` interface: `{ status: number; stdout: string; stderr: string }`
- `runCli(args: string[], cwd?: string): CliResult` — wraps `spawnSync("node", ["--import", "tsx/esm", BIN, ...args], { encoding: "utf-8", cwd: cwd ?? ROOT, env: { ...process.env, FORCE_COLOR: "0", NO_COLOR: "1" } })`. BIN resolves to `join(ROOT, "bin", "qwik.ts")` using `fileURLToPath(new URL("../../..", import.meta.url))` for ROOT. Returns `{ status: result.status ?? -1, stdout: result.stdout ?? "", stderr: result.stderr ?? "" }`.
- `runCreateQwik(args: string[], cwd?: string): CliResult` — same pattern but BIN points to `join(ROOT, "bin", "create-qwik.ts")`. This binary does not exist yet (Phase 6); tests using it will fail with module-not-found, which is the correct red state.

Create `tests/integration/helpers/fixtures.ts` with:
- `FIXTURES_DIR` constant: `fileURLToPath(new URL("../../fixtures", import.meta.url))`
- `setMtimePast(filePath: string, secondsAgo = 3600): void` — uses `utimesSync(filePath, past, past)` where past = `new Date(Date.now() - secondsAgo * 1000)`. Sets both atime and mtime to the same value to avoid OS normalization issues.
- `setMtimeFuture(filePath: string, secondsAhead = 3600): void` — uses `utimesSync(filePath, future, future)` where future = `new Date(Date.now() + secondsAhead * 1000)`.
- `getFixturePath(...segments: string[]): string` — joins FIXTURES_DIR with segments for convenient path construction.

Use `import.meta.url` for all path resolution (not `__dirname`). Import from `node:child_process`, `node:fs`, `node:path`, `node:url`.
